# 2. SQLite永続化ストレージ採用

Date: 2026-02-27
日付: 2026-02-27

## Status

accepted

## 背景

VRChat のログから抽出したハンド履歴と集計結果をローカルで永続化する必要がある。
ユーザー環境で追加のサーバー運用を前提にせず、単一ファイルでバックアップ可能なストレージが求められた。

## 決定

永続化ストレージとして SQLite を採用し、ユーザーデータディレクトリに `vrpoker-stats.db` を保存する。

## 結果

### メリット
- 依存が少なく、インストール不要で利用できる。
- 単一ファイルのためバックアップ・移行が容易。

### トレードオフ
- 大量データでの並列書き込みには限界がある。
- スキーマ変更時はマイグレーション設計が必要。
