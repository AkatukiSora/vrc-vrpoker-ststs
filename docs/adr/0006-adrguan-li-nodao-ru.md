# 6. ADR管理の導入

Date: 2026-02-27
日付: 2026-02-27

## Status

accepted

## 背景

設計判断が暗黙知になりやすく、後から理由や経緯を追いにくい。
OpenCode から全体像を把握しやすい ADR 管理の仕組みが必要だった。

## 決定

`docs/adr/` に ADR を格納し、`adrgen` を使って作成・管理する。
一覧把握は `mise run adr-list` で行い、初期 ADR を整備する。

## 結果

### メリット
- OpenCode や人間が ADR 全体を一括把握できる。
- 置き換え・改訂関係を `adrgen` で追跡できる。

### トレードオフ
- `adrgen` のファイル名がローマ字化される。
- Status/Date は英語表記に合わせる必要がある。
