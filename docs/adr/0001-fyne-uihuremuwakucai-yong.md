# 1. Fyne UIフレームワーク採用

Date: 2026-02-27
日付: 2026-02-27

## Status

accepted

## 背景

デスクトップアプリとして、Windows/Linux で一貫した UI を提供し、ログ表示や統計カードなどを素早く組み立てられる UI フレームワークが必要だった。
Go のコードベースと自然に統合できること、軽量に配布できることが重要だった。

## 決定

Fyne（`fyne.io/fyne/v2`）を UI フレームワークとして採用する。

## 結果

### メリット
- Go で UI を完結できるため、ビルドと配布が簡潔になる。
- クロスプラットフォーム対応で Linux/Windows に同一 UI を提供できる。

### トレードオフ
- Web 技術の UI ライブラリほどの表現力はない。
- Fyne 固有のライフサイクル制約（`fyne.Do` 等）に配慮が必要。
